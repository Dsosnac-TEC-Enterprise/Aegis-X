# Security Policy (SECURITY.md)

## 🛡️ Aegis-X Security Commitment
As a security tool, we take the safety of Aegis-X extremely seriously. We are committed to fixing vulnerabilities that could compromise the operator's host machine, the connected hardware, or the data collected.

## ✅ Supported Versions
Currently, only the latest release is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| v1.2.x  | 🟢 Yes            |
| < v1.1  | ❌ No             |

## 🕵️ Reporting a Vulnerability
**Do not open a public GitHub issue for security vulnerabilities.**

If you discover a security flaw in Aegis-X, please report it via the following steps:
1. Send an email to the maintainer (or use GitHub's Private Vulnerability Reporting feature if enabled).
2. Include a detailed description of the vulnerability and steps to reproduce it (PoC).
3. Give us **90 days** to provide a fix before any public disclosure.

## ⚠️ Disclaimer
Aegis-X is provided "as is." While we strive for maximum security, the maintainers are not responsible for any damage to hardware (Flippers, HackRFs, etc.) or legal repercussions resulting from the use of this software. Use it with caution and respect.
